# 1. Introduction

*(Stub — outline only.)*

This RFC expands the dependency rules of AP-002 (§7) into a complete, enforceable set of allowed and forbidden dependencies across all Units defined by this series.

It restates the AP-001 invariant that dependencies move **toward business meaning** (§8) and applies it to incoming Implementations (AP-003), the Domain (AP-004), Capabilities (AP-005), and Adapter Implementations (AP-007).

# 2. Normative Language

The key words **MUST**, **MUST NOT**, **SHALL**, **SHOULD**, and **MAY** in this document are to be interpreted as described in BCP 14 (RFC 2119 [1] and RFC 8174 [2]) when, and only when, they appear in all capitals, as shown here.

---

# 3. Dependency Direction

*(Outline — to be specified.)*

* All dependencies SHALL move toward business meaning (AP-001 §8).
* The Domain depends on nothing outside itself.

# 5. Layering Rules

*(Outline — to be specified.)*

* Core (Application + Domain + Capabilities) depends on nothing outside itself.
* Implementations may reference the core; never the reverse.
