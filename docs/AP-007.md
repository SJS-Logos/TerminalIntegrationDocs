# 1. Introduction

*(Stub — outline only.)*

This RFC specifies the concrete **Adapter Implementations** that realize the Domain's Capabilities using specific technologies. This is the internal structure of Implementations that AP-002 (§5.3) deliberately deferred.

Adapter Implementations reside outside the core, may reference the core, and MUST NOT be referenced by it (AP-002 §7, AP-006). They contain no business decisions (AP-001 §6.1) and are replaceable without changing the Domain or Application.

# 6. Technology Selections (.NET)

*(Outline — to be specified.)*

The following technology selections apply to services built on the .NET platform. They are **default recommendations**, not mandates: a .NET service SHOULD use them, and MAY use an alternative where justified. They do not bind services built on other platforms (for example C++).

Each selection realizes a Domain Capability (AP-005) and SHALL reside in its own per-technology Adapter Unit (AP-002 §5.3). A selection MUST NOT leak its technology into the core (AP-001 §6.3).

| Capability | Default technology (.NET) | Adapter Unit | Notes |
| --- | --- | --- | --- |
| Persistence | Entity Framework | Persistence Adapter Unit | Realizes the Persistence Capability (AP-005). |
| Messaging | MassTransit | Messaging Adapter Unit | Realizes the Messaging Capability (AP-005). See version note below. |

## 6.1 Persistence — Entity Framework

*(Outline — to be specified.)*

* Entity Framework is the default .NET persistence technology.
* It SHALL be confined to a Persistence Adapter Unit; the core MUST NOT reference it.

## 6.2 Messaging — MassTransit

*(Outline — to be specified.)*

* MassTransit is the default .NET messaging technology.
* Services are pinned to **MassTransit v8 — the last release published under the open-source Apache 2.0 license.** Later releases moved to a commercial license, so services remain on this version.
* It SHALL be confined to a Messaging Adapter Unit; the core MUST NOT reference it.
* Adopting a commercially licensed version is out of scope for this RFC and is tracked separately.
